# ターミナル仕様

## 1. 概要

右ペインに配置するフル機能のターミナルエミュレータ。`portable-pty` で起動した PTY 上にユーザーのシェル（既定は OS のログインシェル）を立ち上げ、その中で `claude` を含む任意の対話型 CLI を動作させる。

実装には `egui_term` をベースに、不足する機能を必要に応じて補強する。

## 2. PTY 起動

| 項目 | 内容 |
|---|---|
| PTY 抽象 | `portable-pty::PtySystem` |
| 起動関数 | `PtySystem::openpty(PtySize)` でマスタ/スレイブ取得後、`CommandBuilder` で子プロセスを spawn |
| 初期サイズ | 右ペインのピクセルサイズ ÷ フォントメトリクスから rows/cols を算出 |

## 3. 起動するシェル

| OS | 既定シェル | フォールバック |
|---|---|---|
| macOS | 環境変数 `$SHELL`（多くは `/bin/zsh`） | `/bin/zsh` → `/bin/bash` |
| Windows | 環境変数 `%COMSPEC%`（多くは PowerShell ではなく `cmd.exe`）ではなく、優先順位を以下に固定 | `pwsh.exe` → `powershell.exe` → `cmd.exe` |

注: Windows の既定は `cmd.exe` よりも `pwsh`/`powershell` を優先する。`claude` の動作確認は PowerShell 系で行われる前提。

シェル切替は **MVP では非対応**（設定ファイルで上書きできるかは設定仕様確定後）。

## 4. 起動時の振る舞い

**決定: B（シェル起動後、自動で `claude` を起動する）**。

起動シーケンス：

1. 既定シェル（3 章）を PTY で起動
2. プロンプトが立ち上がり次第、`claude` を入力して実行
3. ユーザーは即座に Claude Code と対話を開始できる

`claude` 起動の実装方式は実装着手時に確定する候補として：

- (a) シェル起動時の引数で `-c "claude"` 相当を渡す
- (b) PTY マスタへ `claude\n` を書き込む
- (c) シェルに依存せず、PTY 起動の子プロセス自体を `claude` にする

将来的に「claude を起動しない」モードが必要になったら、設定で切り替えられるようにする（MVP 後）。

## 5. ANSI エスケープ対応

VTE パースは `alacritty_terminal` に委譲する。これにより以下に標準対応する。

- カーソル位置制御（CUP / CUU / CUD / CUF / CUB 等）
- 画面・行のクリア（ED / EL）
- 文字属性（SGR）: 太字・下線・反転・取り消し線
- 色: 4-bit / 8-bit / 24-bit（True Color）
- スクロール領域設定（DECSTBM）
- 代替スクリーンバッファ（`vim`/`less`/`claude` の TUI モード等で利用）
- bracketed paste mode

対応しない（または当面後回し）：

- Sixel / iTerm2 画像プロトコル / Kitty 画像プロトコル
- OSC 52（クリップボード経由のリモートコピー）
- マウスレポート（X10 / SGR 1006）は **MVP では対応**（`claude` の対話 UI で必要になる可能性）
- ウィンドウタイトル変更 OSC（OSC 0 / 2）はパースするが、mdpilot のウィンドウタイトルは独自管理（プレビューファイル名）なので**反映しない**

## 6. 選択・コピー・貼り付け

| 操作 | 動作 |
|---|---|
| マウスドラッグ | 連続選択 |
| `Alt`+ドラッグ | 矩形選択（実装可能なら） |
| ダブルクリック | 単語選択 |
| トリプルクリック | 行選択 |
| `Cmd+C`(mac) / `Ctrl+Shift+C`(Win) | 選択範囲を OS クリップボードへコピー |
| `Cmd+V`(mac) / `Ctrl+Shift+V`(Win) | OS クリップボードから PTY へ書き込み |
| `Ctrl+C` | シェル/claude に `Ctrl+C`（SIGINT 相当）を送信 |

`Ctrl+C` と `Cmd+C`/`Ctrl+Shift+C` の使い分けは UI 仕様（`ui.md`）と一致させる。

## 7. スクロールバック

| 項目 | 既定 |
|---|---|
| 行数 | 10,000 行 |
| スクロール操作 | マウスホイール / `Shift+PgUp` / `Shift+PgDn` / `Shift+Home` / `Shift+End` |
| 末尾追従モード | 既定 ON。スクロール位置が末尾にあるとき、新しい出力で自動追従する。スクロールアップすると追従停止 |

行数は設定で変更可（MVP 後）。

## 8. リサイズ

- 右ペインの幅変更・ウィンドウ高さ変更があると、新しい rows/cols を計算して `MasterPty::resize` で PTY に通知
- リサイズ中の頻度を制限するため、デバウンス（例: 50ms）を入れる
- リサイズ通知に対する子プロセスの対応は子側責務（多くの TUI は `SIGWINCH` で再描画）

## 9. フォント

| OS | 既定フォント | サイズ |
|---|---|---|
| macOS | SF Mono | 13pt |
| Windows | Cascadia Mono（フォールバック: Consolas） | 13pt |

- 日本語表示が必要な場合、日本語フォントへのフォールバックを行う（macOS: Hiragino Sans / Windows: Yu Gothic UI）
- フォント・サイズの設定は MVP 後
- 絵文字フォントのフォールバックは要検討（macOS なら Apple Color Emoji が自動でフォールバック）

## 10. 入力・IME

- キー入力は egui のキーイベントから ANSI シーケンスへ変換して PTY に書き込む
- 修飾キー（Shift / Alt / Ctrl / Cmd）の組み合わせを正しく扱う
- **IME（日本語入力）対応は MVP リリース必須要件**。プロンプト入力で日本語を変換・確定でき、変換中の表示が正しく見えること。`egui_term` および egui の IME サポート状況を実装着手時に検証し、不十分であれば自前で補強する（egui の `Event::Ime` ハンドリング、プリエディット文字列の表示、確定文字列の PTY 書き込み）

## 11. 子プロセスの終了

- シェルが終了したとき、ターミナルに「[Process exited with code N]」のような表示を出す
- アプリ起動中に再度シェルを立ち上げる手段は MVP 後に検討
- アプリ終了時には PTY を閉じ、子プロセスに `SIGHUP` を送る

## 12. 既知の制限

- Sixel / 画像プロトコル非対応のため、`claude` がインライン画像を出してきても表示できない
- リモート SSH 経由の高頻度出力ではフレーム落ちの可能性
- 一部 TUI（`tmux` のセッション分割等）は `alacritty_terminal` のスクロール領域・代替スクリーン対応に依存
