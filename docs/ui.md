# UI 仕様

## 1. ウィンドウ構成

```
┌─────────────────────────────────────────────────────────────────┐
│ mdpilot - <プレビューファイル名>                           [_][□][×] │
├──────────────────────────────────┬──────────────────────────────┤
│                                  │                              │
│                                  │                              │
│   左ペイン                       │   右ペイン                   │
│   Markdown プレビュー            ║   Claude チャット             │
│                                  │   (claude と JSON で対話)     │
│                                  │                              │
│                                  │                              │
│                                  │                              │
│                                  │                              │
└──────────────────────────────────┴──────────────────────────────┘
                                  ↑
                          リサイズ可能な境界
```

- 単一ウィンドウ・2 ペイン構成
- 左ペイン: Markdown プレビュー（詳細は `preview.md`）
- 右ペイン: Claude チャット UI（詳細は `chat.md`）
- 中央の境界はマウスドラッグでリサイズ可能

## 2. ウィンドウの既定値

| 項目 | 値 |
|---|---|
| 初期幅 | 1400 px |
| 初期高さ | 900 px |
| 最小幅 | 800 px |
| 最小高さ | 500 px |
| ペイン初期比率 | プレビュー : チャット = 1 : 1 |
| 起動位置 | OS のデフォルト（前回位置の復元は MVP 後） |

数値は MVP の暫定値。設定で上書きできるかは設定ファイル仕様の確定後に決定。

## 3. ペイン操作

| 操作 | 動作 |
|---|---|
| 境界をマウスドラッグ | 左右ペイン幅を変更 |
| 境界をダブルクリック | ペイン比率を 1 : 1 にリセット |
| ペイン最小幅 | 各ペイン 240 px。これ以下に縮められない |

## 4. ウィンドウタイトル

書式: `mdpilot - <プレビュー中ファイルのファイル名>`

- プレビュー対象未指定時: `mdpilot`
- ファイルが未保存変更を持つ概念は存在しない（手動編集機能を持たないため `*` などの印は付けない）

## 5. メニュー

UI の表示言語は **日本語**。メニュー項目・ダイアログ・ステータス表示はすべて日本語で記述する。

### 5.1 macOS（メニューバー）

| メニュー | 項目 | アクション |
|---|---|---|
| mdpilot | mdpilot について | バージョン情報ダイアログ |
|  | 環境設定... | 設定ファイルを既定エディタで開く（MVP 後） |
|  | mdpilot を終了 (`Cmd+Q`) | 終了 |
| ファイル | 開く... (`Cmd+O`) | ファイル選択ダイアログ。選んだ .md をプレビュー対象に設定 |
| 表示 | プレビューを再読込 (`Cmd+R`) | プレビュー強制再読込 |
|  | 拡大 (`Cmd+=`) / 縮小 (`Cmd+-`) | 表示倍率変更（MVP 後） |
| ウインドウ | しまう (`Cmd+M`) | OS 標準 |
|  | 拡大/縮小 | OS 標準 |
| ヘルプ | mdpilot について / GitHub | リンク |

### 5.2 Windows

メニューバーは MVP では持たない（egui のフレーム内に最小限の操作を配置）。代替として：

- ウィンドウ上部に小さなツールバー（開く / 再読込 / 情報）を置く
- もしくは右クリックでコンテキストメニュー

詳細は MVP 実装着手時に決定。

## 6. キーバインド

### 6.1 フォーカスペインによる解釈の切替

チャット・プレビュー・ファイルツリー・モーダル (fuzzy finder / 履歴) では同じキーが別の意味を持つため、**フォーカスのあるペインによってキー解釈を変える** ことを原則とする。Phase 10 で preview に vim モードが入って以降、ペイン非フォーカス時のキーは vim engine に流れる (`feed_vim_events`)。

#### 共通

| キー | 動作 |
|---|---|
| `Cmd+C` (mac) / `Ctrl+C` (Win) | 選択範囲をコピー (preview / chat 両方で OS クリップボードへ) |

#### チャット (入力欄フォーカス時)

| キー | 動作 |
|---|---|
| `Enter` | 入力欄が空でないときに送信 (in_flight でも可。10.25 参照) |
| `Shift+Enter` | 入力欄に改行を挿入 |
| `Esc` | 応答中なら turn を abort (Phase 10.14) |
| `Cmd+V` / `Ctrl+V` | 入力欄に貼り付け |

#### プレビュー (vim Normal モード) — Phase 10.1〜

| キー | 動作 |
|---|---|
| `h`/`j`/`k`/`l` | カーソル移動 (左/下/上/右) |
| `0` / `$` | 行頭 / 行末 |
| `gg` / `G` | バッファ先頭 / 末尾 |
| `w`/`b`/`e` | 単語前進 / 後退 / 単語末尾 (CJK 対応) |
| `i`/`a`/`I`/`A` | Insert に入る (それぞれ位置: 現在/直後/行頭/行末) |
| `o`/`O` | 下/上に新規行を作って Insert に入る |
| `x` | 1 文字削除 (charwise yank → クリップボード) |
| `dd` / `yy` | 行削除 / 行ヤンク (linewise) |
| `d` (Visual) / `y` (Visual) | 選択削除 / 選択ヤンク (クリップボードにも書く) |
| `p` | charwise paste (カーソル直後) |
| `u` / `Ctrl+R` | undo / redo |
| `v` / `V` | Visual / VisualLine モードに入る (Phase 10.6 / 10.6.x) |
| `Y` (Normal, 選択なし) | 現在ファイルの `@<相対パス>\n` を chat input に挿入 + chat フォーカス + カーソル末尾移動 (Phase 10.17 / 10.23) |
| `Y` (Visual / VisualLine) | 選択範囲を引用ブロック化して chat input に append + chat フォーカス (Phase 10.6) |
| `/` → 文字列 + `Enter` | バッファ内検索 (Phase 10.6) |
| `n` / `N` | 次 / 前のマッチへジャンプ |
| `Esc` | Insert/Visual から Normal に戻る |

#### ファイルツリー (フォーカス時) — Phase 10.9

| キー | 動作 |
|---|---|
| `j` / `k` (`↓`/`↑` も可) | エントリ移動 |
| `Space` | ディレクトリの展開/折りたたみ |
| `Enter` | ファイルを開く |
| `Esc` | preview にフォーカスを戻す |

#### モーダル — fuzzy finder (`Cmd+P`) / 履歴ピッカー

| キー | 動作 |
|---|---|
| 文字入力 | クエリ (fuzzy finder) / 表示中フィルタ (履歴) |
| `↑`/`↓`、`Cmd+P`/`Cmd+N` (fuzzy)、`j`/`k` (履歴) | 候補移動 |
| `Enter` | 選択して開く / resume |
| `Esc` | modal を閉じる |

### 6.2 アプリ全体のキーバインド

| キー | アクション |
|---|---|
| `Cmd+Q` (mac) / `Alt+F4` (Win) | アプリ終了 |
| `Cmd+O` (mac) / `Ctrl+O` (Win) | ファイル選択ダイアログを開く (single file pick) |
| `Cmd+R` (mac) / `Ctrl+R` (Win) | プレビュー強制再読込 |
| `Cmd+\` (mac) / `Ctrl+\` (Win) | ペイン境界を 1 : 1 にリセット |
| `Cmd+B` | ファイルツリーの開閉 (開いた直後は tree にフォーカス) |
| `Cmd+P` | fuzzy file finder modal を開く (Phase 10.11) |
| `Cmd+T` | 新規タブを開く |
| `Cmd+W` | 現在タブを閉じる |
| `Cmd+1`〜`Cmd+9` | N 番目のタブにジャンプ |
| `Cmd+Shift+[` / `Cmd+Shift+]` | タブを左 / 右へサイクル (Phase 10.12) |
| `Cmd+Shift+H` / `Cmd+Shift+L` | ペインフォーカスを左 / 右へサイクル (Tree → Preview → Chat, Phase 10.20) |

カスタマイズは MVP 後。

## 7. フォーカスの初期状態

- 起動時は **右ペインのチャット入力欄にフォーカス**。すぐに最初の指示を打ち始められる
- ペインのクリックでフォーカス移動
- 入力欄は `egui::TextEdit::multiline`。IME（日本語入力）は `egui::TextEdit` の標準サポートに依存（`requirements.md` N-07）
- `Tab` キーでフォーカス循環するかは MVP では実装しない（混乱を招くため）

## 8. テーマ

- 既定: **OS のダークモード設定に追従**（`eframe::App` の `dark_mode_or_system` 相当）
- 強制的にライト/ダークを選ぶオプションは MVP 後
- プレビュー側の Markdown スタイルもテーマに連動する（背景色・コードブロック背景色）
- チャット側のメッセージ背景・collapsible ブロック枠もテーマに連動

## 9. ステータス表示

ウィンドウ下端にステータスバーを置くかは未確定。最低限の以下情報は表示したい：

- プレビュー対象ファイルのパス（フル）
- ファイル監視の状態（監視中 / エラー）
- claude 子プロセスの状態（接続中 / 切断 / リトライ中）
- 直近のエラー（あれば）

MVP では「左ペイン上部の小さなパスバー」+「右ペイン下部の claude 接続ステータス」+「エラー時のみトースト」程度で始める。
