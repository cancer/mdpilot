# 要件定義

## 1. 概要

mdpilot は、Claude Code との協調作業によって Markdown ドキュメントを書くためのネイティブ GUI アプリケーションである。ユーザーは内蔵ターミナル内で `claude` を対話的に動かし、生成・編集された Markdown を左ペインのプレビューで確認する。

## 2. ターゲットユーザー

- Markdown でドキュメントを書く開発者・テクニカルライター
- ドキュメントの執筆を Claude Code に任せ、自身は方針指示と確認に専念したい人
- macOS または Windows デスクトップで作業する人

## 3. ユーザーストーリー

1. mdpilot を起動する → 左ペインは空、右ペインに内蔵ターミナルが立ち上がる
2. 右ペインのターミナルで `claude` を起動し、`README を書いて` のように指示する
3. Claude Code がファイル（例: `README.md`）を編集する
4. 左ペインのプレビューに編集結果が自動反映される
5. ユーザーは仕上がりを見て、ターミナル側で次の指示（修正・追記）を Claude Code に出す
6. これを繰り返してドキュメントを完成させる

## 4. 機能要件

### 4.1 MVP（必須）

| ID | 要件 | 備考 |
|---|---|---|
| F-01 | アプリ起動時に 2 ペイン構成のウィンドウを表示する（左: プレビュー / 右: ターミナル） | ペイン境界はマウスでリサイズ可能 |
| F-02 | 右ペインに PTY ベースのフル機能ターミナルを内蔵する | `claude` を含む対話型 CLI が動作すること。起動時に既定シェルを上げた後、自動で `claude` を実行する |
| F-03 | ターミナルは ANSI エスケープシーケンス（色・カーソル制御）に対応する | 詳細は `terminal.md` |
| F-04 | ターミナルでの選択・コピー・貼り付けが動作する | OS のクリップボード経由 |
| F-05 | ターミナルにスクロールバックが存在する | 行数の上限は設定可能 |
| F-06 | 左ペインで CommonMark + GFM 拡張（テーブル・チェックボックス・取り消し線）の Markdown をレンダリングする | 詳細は `preview.md` |
| F-07 | コードブロックのシンタックスハイライトを表示する | |
| F-08 | プレビュー対象のファイルが外部から変更されたら、自動で再読込・再レンダリングする | ファイル監視を使う |
| F-09 | Claude Code 側からプレビュー対象ファイルを開ける仕組みを提供する | 実現方式は未確定（`claude-integration.md` で検討） |
| F-10 | macOS（Apple Silicon / Intel）と Windows 10/11 でビルドできる | |

## 5. 非機能要件

| ID | 要件 |
|---|---|
| N-01 | 起動から操作可能になるまで 3 秒以内（リリースビルド、SSD ストレージ前提） |
| N-02 | ターミナル入力の表示遅延がユーザーに知覚できないこと（目安 50ms 以下） |
| N-03 | Claude Code のストリーミング出力に対してフレーム落ちなく追従すること |
| N-04 | プレビューの再レンダリングは Markdown 1 万字程度なら 100ms 以内 |
| N-05 | アプリの異常終了でユーザーの編集データを失わないこと。アプリは編集を行わないため、データは Claude Code 側および OS ファイルシステムに依存する |
| N-06 | リリースバイナリは macOS / Windows ともに単一実行可能ファイルとして配布できる |
| N-07 | ターミナルで IME（日本語入力）が安定動作すること。変換中の表示・確定文字列の入力が正しく行えること |

## 6. スコープ外

明示的に「やらない」と決めた事項。

- 手動でのテキスト編集（プレビュー専用）
- Linux サポート（当面）
- ファイラー機能・プロジェクト管理
- WYSIWYG 編集
- プラグインシステム
- 自動アップデート機構
- App Store / Microsoft Store への登録
- 同期サービス・クラウド連携
- Claude Code 以外のエージェント（自前で API を叩く等）の組み込み
- アクセシビリティ対応（スクリーンリーダー対応、コントラスト基準など）

## 7. 制約

| 制約 | 内容 |
|---|---|
| 言語 | Rust |
| GUI | egui (eframe) |
| ターミナル | egui_term + alacritty_terminal + portable-pty |
| Markdown | egui_commonmark（暫定。最終確認は実装時） |
| 動作前提 | ユーザーが `claude` コマンドを `PATH` 上に持っていること |
| UI 表示言語 | 日本語（メニュー・ダイアログ・ステータス表示） |
