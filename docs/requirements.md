# 要件定義

## 1. 概要

mdpilot は、Claude Code との協調作業によって Markdown ドキュメントを書くためのネイティブ GUI アプリケーションである。ユーザーは右ペインの内蔵 chat UI からプロンプトを送り、生成・編集された Markdown を左ペインのプレビューで確認する。

mdpilot は `claude` CLI を非対話モード（`--print --input-format=stream-json --output-format=stream-json`）で子プロセスとして spawn し、JSON ストリームでメッセージを送受信する。ターミナルエミュレータは持たない。

## 2. ターゲットユーザー

- Markdown でドキュメントを書く開発者・テクニカルライター
- ドキュメントの執筆を Claude Code に任せ、自身は方針指示と確認に専念したい人
- macOS または Windows デスクトップで作業する人

## 3. ユーザーストーリー

1. mdpilot を起動する → 左ペインは空、右ペインに chat UI が立ち上がる
2. 右ペインの入力欄に `README を書いて` のように指示を入力して送信
3. mdpilot が背後で `claude` を spawn し、応答ストリームを右ペインに表示する
4. Claude Code がファイル（例: `README.md`）を編集する
5. 左ペインのプレビューに編集結果が自動反映される
6. ユーザーは仕上がりを見て、右ペインで次の指示（修正・追記）を Claude Code に出す
7. これを繰り返してドキュメントを完成させる
8. 次回起動時は同じセッションを `--continue` で再開できる

## 4. 機能要件

### 4.1 MVP（必須）

| ID | 要件 | 備考 |
|---|---|---|
| F-01 | アプリ起動時に 2 ペイン構成のウィンドウを表示する（左: プレビュー / 右: chat UI） | ペイン境界はマウスでリサイズ可能 |
| F-02 | 右ペインに Claude Code 連携 chat UI を内蔵する | `claude --print --input-format=stream-json --output-format=stream-json --dangerously-skip-permissions` を子プロセスとして spawn し、JSON で双方向にメッセージをやり取りする。詳細は `chat.md` |
| F-03 | claude のレスポンスをストリーミングで右ペインに表示する | テキスト・思考（thinking）・ツール呼び出し・ツール結果のイベントを順次描画 |
| F-04 | ツール呼び出し（Edit / Write / Bash 等）を **折りたたみ可能なブロック**として表示する | ツール名 + 入力 + 出力。既定では折りたたみ、クリックで展開 |
| F-05 | chat 履歴のスクロール、メッセージ選択・コピーが動作する | OS のクリップボード経由 |
| F-06 | 左ペインで CommonMark + GFM 拡張（テーブル・チェックボックス・取り消し線）の Markdown をレンダリングする | 詳細は `preview.md` |
| F-07 | コードブロックのシンタックスハイライトを表示する | |
| F-08 | プレビュー対象のファイルが外部から変更されたら、自動で再読込・再レンダリングする | ファイル監視を使う |
| F-09 | Claude Code が書いた／編集したファイルをプレビュー対象に自動切替する | claude-integration.md 5 章「自動追従」案 A |
| F-10 | macOS（Apple Silicon / Intel）と Windows 10/11 でビルドできる | |
| F-11 | プロジェクトルートごとに claude セッション ID を 1 つ保存し、次回起動時に `--continue` 相当で再開する | 詳細は `chat.md` および `claude-integration.md` |

## 5. 非機能要件

| ID | 要件 |
|---|---|
| N-01 | 起動から操作可能になるまで 3 秒以内（リリースビルド、SSD ストレージ前提） |
| N-02 | chat 入力 → claude 応答開始までの遅延が UI 上で知覚できる範囲（ネットワーク + Claude API の応答時間に支配される） |
| N-03 | claude のストリーミング出力に対してフレーム落ちなく追従すること |
| N-04 | プレビューの再レンダリングは Markdown 1 万字程度なら 100ms 以内 |
| N-05 | アプリの異常終了でユーザーの編集データを失わないこと。**mdpilot は `.md` ファイルへの書込み経路を持たない**（編集はすべて claude 子プロセス経由）ため、データは Claude Code 側および OS ファイルシステムに依存する |
| N-06 | リリースバイナリは macOS / Windows ともに単一実行可能ファイルとして配布できる |
| N-07 | chat 入力欄で IME（日本語入力）が安定動作すること。`egui::TextEdit` の IME サポートに依存。変換中の表示・確定文字列の送信が正しく行えること |

## 6. スコープ外

明示的に「やらない」と決めた事項。

- 手動でのテキスト編集（プレビュー専用、編集は claude 経由のみ）
- 内蔵ターミナルエミュレータ（ANSI / PTY / シェル起動などは持たない。`claude` CLI を spawn するのみ）
- Linux サポート（当面）
- ファイラー機能・プロジェクト管理
- WYSIWYG 編集
- プラグインシステム
- 自動アップデート機構
- App Store / Microsoft Store への登録
- 同期サービス・クラウド連携
- Claude Code 以外のエージェント（自前で Anthropic API を直接叩く、別 SDK 統合等）の組み込み
- アクセシビリティ対応（スクリーンリーダー対応、コントラスト基準など）

## 7. 制約

| 制約 | 内容 |
|---|---|
| 言語 | Rust |
| GUI | egui (eframe) |
| Markdown | egui_commonmark（暫定。最終確認は実装時） |
| Claude 連携 | `claude` CLI（`--print --input-format=stream-json --output-format=stream-json`） |
| 動作前提 | ユーザーが `claude` コマンドを `PATH` 上に持っていること、かつ Claude Code に認証済みであること |
| UI 表示言語 | 日本語（メニュー・ダイアログ・ステータス表示） |
| パーミッション | MVP は `--dangerously-skip-permissions` 起動でツール許可ダイアログをスキップする。F-28 で MVP 後に安全モードを追加 |
